# 앱 책임 범위

## order-matching이 하는 일

### 1. 상품 매칭 규칙 관리

- 매칭 생성: variant ↔ [SKU, 수량] 대응 등록
- 매칭 수정: SKU 변경, 수량 변경, strategy 변경
- 매칭 삭제
- 매칭 조회: variant별, master별, strategy별

### 2. 주문 변환

- 판매주문(variant 기반)을 받아 매칭 규칙을 적용, 재고주문(SKU 기반)으로 변환
- 변환 시점의 매칭 규칙을 스냅샷으로 보존 (매칭이 나중에 변경되어도 기존 주문에 영향 없음)
- 변환 결과를 이벤트로 발행

### 3. 미매칭 현황 제공

- PIM의 variant 목록과 매칭 레코드를 비교하여 미매칭 variant를 도출
- 관리자 대시보드에서 "매칭이 필요한 상품" 목록을 조회할 수 있도록 지원

---

## order-matching이 하지 않는 일

### variant/SKU 존재 검증

매칭 앱은 variant나 SKU의 실제 존재 여부를 검증하지 않는다. 들어오는 이벤트나 관리자 요청을 신뢰한다.

- variant가 존재하는지는 PIM의 책임
- SKU가 존재하는지는 WMS의 책임
- 매칭 앱은 전달받은 ID를 그대로 저장

### 재고주문 상태 관리

재고주문의 lifecycle(출고 진행, 부분 출고, 완료 등)은 WMS가 관리한다. 매칭 앱은 변환까지만 책임지고, 이후 이행 상태는 관여하지 않는다.

### 재고 관리

재고 수량, 재고 이벤트 소싱, 입출고 등은 WMS의 고유 책임이다.

### 판매상품 관리

variant의 옵션 구조, 승인 워크플로우, 가격 등은 PIM의 고유 책임이다.

## 다른 앱과의 역할 경계

```
┌───────────────────────────────────────────────────────────┐
│                    order-matching                         │
│                                                          │
│  [상품 매칭 규칙]          [주문 변환]                      │
│  variant ↔ SKU 대응        판매주문 → 재고주문              │
│                                                          │
└──────────────┬──────────────────────┬─────────────────────┘
               │                      │
    이벤트 소비/발행            이벤트 소비/발행
               │                      │
       ┌───────┴───────┐      ┌───────┴───────┐
       │               │      │               │
   ┌───▼───┐     ┌─────▼──┐  │         ┌─────▼──────┐
   │  PIM  │     │  WMS   │  │         │ 판매채널    │
   │       │     │        │  │         │ (channel-  │
   │variant│     │SKU     │  │         │  adapter)  │
   │SoT    │     │SoT     │  │         │            │
   │       │     │재고주문 │  │         │판매주문    │
   │       │     │SoT     │  │         │SoT         │
   └───────┘     └────────┘  │         └────────────┘
                              │
                        관리자 수동 주문도
                        WMS에 직접 등록 가능
```

### SoT 정리

| 데이터 | SoT |
|--------|-----|
| 판매상품 (variant, 옵션) | PIM |
| 재고상품 (SKU) | WMS |
| 상품 매칭 규칙 (variant ↔ SKU) | **order-matching** |
| 판매주문 | 각 판매채널 (channel-adapter, 관리자 등) |
| 재고주문 | WMS |
| 주문 변환 (판매주문 → 재고주문 매핑 기록) | **order-matching** |
