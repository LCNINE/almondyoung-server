# 출고주문 생성은 재고예약과 분리한다

Core 의 출고주문 생성은 판매주문 라인을 SKU 구성으로 번역하는 단계이며, 판매상품↔재고상품 매칭이 없으면 생성 자체가 불가능하다. 반면 매칭이 있고 실제 재고가 부족한 경우에는 출고주문과 출고주문 아이템을 생성한 뒤 reservation 단계에서 실패/대기 상태로 남긴다. 재고 부족을 출고주문 생성 실패로 처리하면 결제 수락된 판매주문이 왜 출고되지 못하는지 운영자가 추적할 객체가 사라진다.

## Consequences

- Canonical 출고주문 생성 경로는 매칭 검증을 fail-closed 로 수행해야 한다.
- Reservation 은 출고주문 생성 후 상태 전이이며, 재고 부족은 출고 불가/대기 상태와 실패 사유로 표현한다.
- 생성과 예약을 한 트랜잭션 성공 조건으로 묶는 API 는 폐기하거나 리팩터링한다.
- 빈 출고주문 또는 매칭 누락 라인을 조용히 제외한 출고주문은 만들지 않는다.
