# Product Purchase Constraint Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Core Catalog purchase constraint data for product master versions, publish it in active product snapshots, and project it to Medusa product metadata without implementing admin-web UI or Medusa checkout enforcement.

**Architecture:** Model purchase constraints like other version-owned catalog child data: entity row plus version mapping, draft mapping copy, copy-on-write on shared-row edits, and orphan cleanup after reference-count checks. Core Catalog owns the constraint data but does not validate orders; channel-adapter receives active-version snapshots and projects `pimPurchaseConstraint` into Medusa product metadata for future local enforcement.

**Tech Stack:** NestJS, Drizzle ORM, PostgreSQL, Jest, Zod event contracts, channel-adapter Medusa Admin SDK payload transformer.

```ts
import { BadRequestException, Injectable, NotFoundException } from '@nestjs/common';
import { DbService, InjectDb } from '@app/db';
import { and, count, eq, ne } from 'drizzle-orm';
import { v7 as uuidv7 } from 'uuid';
import {
  type PimSchema,
  productMasterPurchaseConstraints,
  productMasterVersions,
  productPurchaseConstraints,
} from '../../../schema/catalog.schema';
import type { DbTransaction, PurchaseConstraintReadModel } from '../../../catalog.types';
import { UpsertPurchaseConstraintDto } from '../dto/purchase-constraints';

@Injectable()
export class ProductPurchaseConstraintsService {
  constructor(@InjectDb() private readonly db: DbService<PimSchema>) {}

  private get client() {
    return this.db.db;
  }

  private async inTx<T>(fn: (tx: DbTransaction) => Promise<T>, tx?: DbTransaction): Promise<T> {
    return tx ? fn(tx) : this.client.transaction(fn);
  }

  isDeleteIntent(input: UpsertPurchaseConstraintDto): boolean {
    return input.requiresMembership === false && (input.lifetimeQuantityLimit ?? null) === null;
  }

  assertValidInput(input: UpsertPurchaseConstraintDto): void {
    if (input.lifetimeQuantityLimit !== null && input.lifetimeQuantityLimit !== undefined && input.lifetimeQuantityLimit <= 0) {
      throw new BadRequestException('lifetimeQuantityLimit must be a positive integer or null');
    }
  }

  async getForVersion(masterId: string, versionId: string, tx?: DbTransaction): Promise<PurchaseConstraintReadModel | null> {
    return this.inTx(async (tx) => {
      await this.assertVersionBelongsToMaster(masterId, versionId, tx);
      const [row] = await tx
        .select({
          id: productPurchaseConstraints.id,
          requiresMembership: productPurchaseConstraints.requiresMembership,
          lifetimeQuantityLimit: productPurchaseConstraints.lifetimeQuantityLimit,
        })
        .from(productMasterPurchaseConstraints)
        .innerJoin(
          productPurchaseConstraints,
          eq(productMasterPurchaseConstraints.purchaseConstraintId, productPurchaseConstraints.id),
        )
        .where(
          and(
            eq(productMasterPurchaseConstraints.masterId, masterId),
            eq(productMasterPurchaseConstraints.versionId, versionId),
          ),
        )
        .limit(1);

      return row ?? null;
    }, tx);
  }

  async upsertForDraft(
    masterId: string,
    versionId: string,
    input: UpsertPurchaseConstraintDto,
    tx?: DbTransaction,
  ): Promise<PurchaseConstraintReadModel | null> {
    return this.inTx(async (tx) => {
      this.assertValidInput(input);
      await this.assertDraftVersion(masterId, versionId, tx);

      if (this.isDeleteIntent(input)) {
        await this.deleteForDraft(masterId, versionId, tx);
        return null;
      }

      const existing = await this.getMapping(masterId, versionId, tx);
      if (!existing) {
        const [created] = await tx
          .insert(productPurchaseConstraints)
          .values({
            id: uuidv7(),
            requiresMembership: input.requiresMembership,
            lifetimeQuantityLimit: input.lifetimeQuantityLimit ?? null,
            createdAt: new Date(),
            updatedAt: new Date(),
          })
          .returning();

        await tx.insert(productMasterPurchaseConstraints).values({
          id: uuidv7(),
          masterId,
          versionId,
          purchaseConstraintId: created.id,
          createdAt: new Date(),
        });

        return {
          id: created.id,
          requiresMembership: created.requiresMembership,
          lifetimeQuantityLimit: created.lifetimeQuantityLimit,
        };
      }

      const isShared = await this.isConstraintShared(existing.purchaseConstraintId, versionId, tx);
      if (isShared) {
        const [created] = await tx
          .insert(productPurchaseConstraints)
          .values({
            id: uuidv7(),
            requiresMembership: input.requiresMembership,
            lifetimeQuantityLimit: input.lifetimeQuantityLimit ?? null,
            createdAt: new Date(),
            updatedAt: new Date(),
          })
          .returning();

        await tx
          .update(productMasterPurchaseConstraints)
          .set({ purchaseConstraintId: created.id })
          .where(eq(productMasterPurchaseConstraints.id, existing.mappingId));

        await this.deleteIfOrphan(existing.purchaseConstraintId, tx);

        return {
          id: created.id,
          requiresMembership: created.requiresMembership,
          lifetimeQuantityLimit: created.lifetimeQuantityLimit,
        };
      }

      const [updated] = await tx
        .update(productPurchaseConstraints)
        .set({
          requiresMembership: input.requiresMembership,
          lifetimeQuantityLimit: input.lifetimeQuantityLimit ?? null,
          updatedAt: new Date(),
        })
        .where(eq(productPurchaseConstraints.id, existing.purchaseConstraintId))
        .returning();

      return {
        id: updated.id,
        requiresMembership: updated.requiresMembership,
        lifetimeQuantityLimit: updated.lifetimeQuantityLimit,
      };
    }, tx);
  }

  async deleteForDraft(masterId: string, versionId: string, tx?: DbTransaction): Promise<void> {
    return this.inTx(async (tx) => {
      await this.assertDraftVersion(masterId, versionId, tx);
      const existing = await this.getMapping(masterId, versionId, tx);
      if (!existing) return;

      await tx
        .delete(productMasterPurchaseConstraints)
        .where(eq(productMasterPurchaseConstraints.id, existing.mappingId));

      await this.deleteIfOrphan(existing.purchaseConstraintId, tx);
    }, tx);
  }

  async copyMapping(masterId: string, fromVersionId: string, toVersionId: string, tx: DbTransaction): Promise<void> {
    const existing = await this.getMapping(masterId, fromVersionId, tx);
    if (!existing) return;

    await tx.insert(productMasterPurchaseConstraints).values({
      id: uuidv7(),
      masterId,
      versionId: toVersionId,
      purchaseConstraintId: existing.purchaseConstraintId,
      createdAt: new Date(),
    });
  }

  private async assertVersionBelongsToMaster(masterId: string, versionId: string, tx: DbTransaction) {
    const [version] = await tx
      .select({ id: productMasterVersions.id, status: productMasterVersions.status })
      .from(productMasterVersions)
      .where(and(eq(productMasterVersions.id, versionId), eq(productMasterVersions.masterId, masterId)))
      .limit(1);

    if (!version) {
      throw new NotFoundException(`Version ${versionId} not found for master ${masterId}`);
    }

    return version;
  }

  private async assertDraftVersion(masterId: string, versionId: string, tx: DbTransaction): Promise<void> {
    const version = await this.assertVersionBelongsToMaster(masterId, versionId, tx);
    if (version.status !== 'draft') {
      throw new BadRequestException('Only draft versions can be modified');
    }
  }

  private async getMapping(masterId: string, versionId: string, tx: DbTransaction) {
    const [mapping] = await tx
      .select({
        mappingId: productMasterPurchaseConstraints.id,
        purchaseConstraintId: productMasterPurchaseConstraints.purchaseConstraintId,
      })
      .from(productMasterPurchaseConstraints)
      .where(
        and(
          eq(productMasterPurchaseConstraints.masterId, masterId),
          eq(productMasterPurchaseConstraints.versionId, versionId),
        ),
      )
      .limit(1);

    return mapping ?? null;
  }

  private async isConstraintShared(constraintId: string, currentVersionId: string, tx: DbTransaction): Promise<boolean> {
    const [result] = await tx
      .select({ total: count() })
      .from(productMasterPurchaseConstraints)
      .where(
        and(
          eq(productMasterPurchaseConstraints.purchaseConstraintId, constraintId),
          ne(productMasterPurchaseConstraints.versionId, currentVersionId),
        ),
      );

    return Number(result?.total ?? 0) > 0;
  }

  private async deleteIfOrphan(constraintId: string, tx: DbTransaction): Promise<void> {
    const [result] = await tx
      .select({ total: count() })
      .from(productMasterPurchaseConstraints)
      .where(eq(productMasterPurchaseConstraints.purchaseConstraintId, constraintId));

    if (Number(result?.total ?? 0) === 0) {
      await tx.delete(productPurchaseConstraints).where(eq(productPurchaseConstraints.id, constraintId));
    }
  }
}
```

## Acceptance Criteria

- Core has `product_purchase_constraints` and `product_master_purchase_constraints`.
- A product master version can have zero or one purchase constraint.
- Zero mapped rows means no purchase constraint.
- `lifetimeQuantityLimit` accepts `null` or positive integers only.
- Draft creation copies the purchase-constraint mapping.
- Editing a draft purchase constraint performs CoW when the mapped row is shared with another version.
- Deleting or disabling all constraint fields removes the mapping and deletes the constraint row only when no version still references it.
- Version detail responses include `purchaseConstraint`.
- Active publish/rollback snapshots include optional `purchaseConstraint`.
- Draft edits do not emit purchase-constraint-specific sales-channel events.
- channel-adapter projects `snapshot.purchaseConstraint` into Medusa product metadata as `pimPurchaseConstraint`.
- Legacy `metadata.isMembershipOnly` remains separate.
- Medusa checkout enforcement and admin-web editing UI are not included in this implementation slice.

- Type consistency: The canonical property names are `purchaseConstraint`, `requiresMembership`, `lifetimeQuantityLimit`, and `pimPurchaseConstraint` throughout the plan.
