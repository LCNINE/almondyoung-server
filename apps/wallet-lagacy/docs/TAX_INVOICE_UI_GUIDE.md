G# 🧾 [Feature] 세금계산서 발행 관리 어드민 UI 구현

## 📋 Overview

운영자가 세금계산서 발행 요청 내역을 조회/관리하고, 홈텍스 대량 업로드용 엑셀을 다운로드할 수 있는 백오피스 화면을 구현합니다.

**Scope:**

- ✅ 세금계산서 발행 요청 내역 관리 (내부 장부)
- ✅ 홈텍스 업로드용 엑셀 데이터 생성 및 다운로드
- ❌ 실제 세금계산서 조회/출력 (홈텍스에서만 가능)

### ⚠️ 중요 비즈니스 룰

- 세금계산서 **신청(REQUESTED)**은 구매 확정 전에도 가능
- 실제 **발급 처리 및 홈텍스 업로드**는 **구매 확정된 주문만** 대상
- **실제 세금계산서는 홈텍스에서만 조회 가능** (우리는 발행 내역만 관리)
- 우리 시스템은 홈텍스 업로드를 위한 데이터 준비 및 장부 관리 역할

```
정산/세무 > 세금계산서 관리
```

## ✅ Requirements

### Tab 1: 세금계산서 발행 관리 내역

> 💡 우리 시스템에서 관리하는 발행 요청/처리 내역 (장부)
> 실제 세금계산서 문서는 홈텍스에서만 조회 가능

#### 📌 검색/필터 영역

- [ ] **기간 필터**
  - 라벨: `공급시기`
  - 타입: DateRangePicker
  - 기본값: 최근 30일

- [ ] **상태 필터**
  - 옵션: 전체 / REQUESTED / EXPORTED / ISSUED / ERROR / CANCELLED

- [ ] **검색어 필터**
  - 검색 타입: 주문번호 / 회원 이메일/ID / 사업자등록번호
  - Input 필드

- [ ] **액션 버튼**
  - `검색` (Primary) / `초기화` (Ghost)

#### 📊 리스트 테이블

- [ ] **컬럼 구성**
      | 컬럼명 | 속성 | 비고 |
      |--------|------|------|
      | 체크박스 | - | 추후 bulk 액션 대비 |
      | 처리 상태 | Badge | 색상 코딩 |
      | 주문번호 | Link | 새 탭 이동 |
      | 공급시기 | Date | YYYY-MM-DD |
      | 공급받는자 상호 | Text | businessName |
      | 사업자등록번호 | Text | businessNumber |
      | 공급가액 | Number | 우측 정렬 |
      | 세액 | Number | 우측 정렬 |
      | 합계금액 | Number | 우측 정렬 |
      | 발행요청일 | DateTime | createdAt |
      | 최종 상태변경일 | DateTime | updatedAt |

- [ ] **상태별 색상 코딩**
  - REQUESTED: 회색 (발행 요청됨)
  - EXPORTED: 파랑 (엑셀 추출됨)
  - ISSUED: 초록 (홈텍스 발행 완료)
  - ERROR: 빨강 (처리 실패)
  - CANCELLED: 회색 라인 (취소)

- [ ] **행 클릭 시 상세 패널/모달 열기**

#### 🔍 상세보기 패널

> ℹ️ 발행 데이터 스냅샷 및 처리 상태만 표시 (실제 세금계산서는 홈텍스에서 확인)

- [ ] **헤더**
  - 제목: `발행 내역 상세`
  - 서브: 주문번호 / 발행 요청 ID
  - 처리 상태 배지
  - 안내 텍스트: "실제 세금계산서는 홈텍스에서 확인하실 수 있습니다"

- [ ] **섹션 1: 처리 정보**
  - 주문번호
  - 발행 요청 ID
  - 공급시기(작성일자)
  - 처리 상태
  - 발행 요청일
  - 엑셀 추출일 (EXPORTED 상태인 경우)
  - 홈텍스 업로드 완료일 (ISSUED 상태인 경우)
  - 홈텍스 승인번호 (있는 경우)
  - 구매 확정 여부/일시 (Optional)

- [ ] **섹션 2: 공급받는자 정보 (스냅샷)**
  - 상호
  - 사업자등록번호
  - 대표자명
  - 주소
  - 이메일 (있으면)
  - 업태/종목 (추후 추가 시)

- [ ] **섹션 3: 금액 정보 (스냅샷)**
  - 공급가액
  - 세액
  - 합계금액

- [ ] **섹션 4: 품목 정보 (스냅샷)**
  - 테이블 형태:
    - 품목명, 규격, 수량, 단가, 공급가액
  - 상단 요약 텍스트:
    - `예: 상품 3건, 총 120,000원`

- [ ] **섹션 5: 처리 이력 (Optional)**
  - 상태 변경 이력 타임라인
  - 각 상태 변경 시각 및 사유

- [ ] **섹션 6: 디버그 정보 (Optional)**
  - 토글로 raw snapshot JSON, events 로그 표시
  - 개발자/CS 용도

### Tab 2: 홈텍스 업로드용 엑셀 다운로드

> 🚨 **중요:** 구매 확정된 발급 대상만 조회 (홈텍스에 실제 업로드 가능한 건만)

#### 🎯 페이지 설명

```
제목: 홈텍스 업로드용 엑셀 다운로드

설명: 선택한 기간의 구매 확정된 세금계산서 발급 대상 데이터를 조회하고,
홈텍스 대량 업로드 양식에 맞는 엑셀 파일로 다운로드합니다.
다운로드한 엑셀 파일을 홈텍스에 직접 업로드하여 세금계산서를 발급하세요.

ℹ️ 구매 확정되지 않은 신청 건은 표시되지 않습니다.
ℹ️ 실제 세금계산서 발급 및 조회는 홈텍스에서 진행됩니다.
```

#### 🔎 필터 영역

- [ ] **기준일자**
  - 라벨: `기준일자 (공급시기 기준)`
  - DatePicker / DateRangePicker
  - 기본값: 어제 또는 최근 1일

- [ ] **대상 상태**
  - 라벨: `발급 대상 상태`
  - 기본값: `REQUESTED` (구매 확정된 발행 요청 건)
  - Optional: REQUESTED + EXPORTED 포함 가능
  - 설명 텍스트: "구매 확정된 주문만 조회됩니다"

- [ ] **조회 버튼**
  - 로딩 스피너 + 비활성화 처리

- [ ] **조회 로직**
  - 🔒 **백엔드 API는 구매 확정된 주문만 필터링하여 반환**
  - 구매 확정 전 REQUESTED 건은 제외

#### 📋 조회 결과 테이블

- [ ] **컬럼 구성**
      | 컬럼명 | 속성 | 비고 |
      |--------|------|------|
      | 체크박스 | - | 선택 다운로드용 |
      | 주문번호 | Text | - |
      | 공급받는자 상호 | Text | buyerName |
      | 사업자등록번호 | Text | buyerBusinessNumber |
      | 작성일자 | Date | issueDate |
      | 공급가액 | Number | supplyAmount |
      | 세액 | Number | taxAmount |
      | 합계금액 | Number | totalAmount |
      | 품목 요약 | Text | 예: 상품A 외 2건 |

- [ ] **데이터 제약**
  - 모든 행은 구매 확정된 주문
  - 구매 확정 전 주문은 표시 안 됨

#### 💾 엑셀 다운로드

- [ ] **전체 엑셀 다운로드 버튼**
  - 현재 조회 결과 전체 다운로드
  - (구매 확정된 발급 대상만 포함)

- [ ] **선택 항목 엑셀 다운로드 버튼**
  - 체크박스 선택 항목만 다운로드
  - 선택 없을 시 disabled

- [ ] **파일명 규칙**
  - 단일: `hometax-upload-2025-01-20.xlsx`
  - 범위: `hometax-upload-2025-01-01~2025-01-31.xlsx`

- [ ] **엑셀 생성 로직 (Frontend)**
  1. 백엔드 응답 JSON 배열 (`HometaxExportRow[]`)
  2. 홈텍스 대량 업로드 양식 컬럼 순서로 맵핑
  3. xlsx 라이브러리로 시트 생성
  4. 브라우저 다운로드 트리거

- [ ] **다운로드 후 안내**
  - Toast 메시지: "엑셀 파일을 홈텍스에 업로드하여 세금계산서를 발급하세요"


## 💡 Implementation Notes

1. **발행 관리 내역 (Tab 1)**
   - 검색 + 리스트 + 상세보기 UI
   - 발행 요청/처리 내역 장부 역할
   - 실제 세금계산서는 제공하지 않음 (홈텍스에서만 조회 가능)

2. **엑셀 다운로드 (Tab 2)**
   - 기준일자/상태로 조회 → 결과 테이블 → 엑셀 다운로드
   - **구매 확정된 발급 대상만** 조회 및 다운로드
   - 다운로드한 엑셀을 홈텍스에 업로드하여 실제 발급

3. **엑셀 생성**
   - 백엔드 JSON 배열 기반 프론트엔드 생성
   - 홈텍스 대량 업로드 양식 준수

4. **데이터 범위 차이**
   - Tab 1: 전체 발행 요청/처리 내역 (장부)
   - Tab 2: 구매 확정 + 홈텍스 업로드 가능 건만

5. **시스템 역할**
   - ✅ 발행 데이터 관리 (내부 장부)
   - ✅ 홈텍스 업로드용 데이터 준비
   - ❌ 실제 세금계산서 발급/조회 (홈텍스 담당)

## 📝 Additional Notes

### 운영자 가이드

**세금계산서 발급 프로세스:**

1. 고객이 세금계산서 발행 신청 (구매 확정 전/후 가능)
2. Tab 1에서 발행 요청 내역 확인
3. Tab 2에서 구매 확정된 건만 조회하여 엑셀 다운로드
4. **홈텍스에 엑셀 업로드 → 실제 세금계산서 발급**
5. 발급 완료 후 상태를 ISSUED로 업데이트 (수동 또는 자동)
6. 고객은 홈텍스에서 세금계산서 확인/출력
