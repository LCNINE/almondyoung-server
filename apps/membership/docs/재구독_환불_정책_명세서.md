# 재구독 후 환불 정책 명세서

## 📋 비즈니스 시나리오

### 문제 상황

```
사용자 A:
1. 첫 구독 → 7일 무료 체험 → 취소 (전액 환불)
2. 재구독 (무료 체험 없음, 즉시 결제 9,900원)
3. 10초 후 변심 → 취소 요청
```

**핵심 질문**: 재구독 후 즉시 취소 시 환불을 해줘야 하는가?

---

## 🎯 환불 판단 기준

### 옵션 1: 시간 기반 환불 (추천)

**"재구독 후 N시간 이내 취소 시 전액 환불"**

```typescript
// 예: 24시간 이내 취소 시 전액 환불
if (재구독 && 구독_후_경과시간 < 24시간 && 혜택_사용_없음) {
  return 전액_환불;
}
```

**장점:**

- 사용자 친화적 (변심 허용)
- 명확한 정책
- 구현 간단

**단점:**

- 악용 가능성 (24시간 동안 혜택 사용 후 취소)

---

### 옵션 2: 혜택 사용 여부 기반 환불 (공정함)

**"재구독 후 멤버십 혜택을 사용하지 않았다면 전액 환불"**

```typescript
if (재구독 && 혜택_사용_없음) {
  return 전액_환불;
} else if (재구독 && 혜택_사용_있음) {
  return 환불_불가_또는_부분환불;
}
```

**장점:**

- 공정함 (혜택 사용 안 했으면 환불)
- 악용 방지

**단점:**

- 혜택 사용 여부 추적 필요
- MSA 간 통신 필요

---

### 옵션 3: 하이브리드 (시간 + 혜택)

**"재구독 후 24시간 이내 + 혜택 미사용 시 전액 환불"**

```typescript
if (재구독 && 구독_후_경과시간 < 24시간 && 혜택_사용_없음) {
  return 전액_환불;
} else if (재구독 && 구독_후_경과시간 < 24시간 && 혜택_사용_있음) {
  return 부분_환불; // 사용한 혜택만큼 차감
} else {
  return 환불_불가;
}
```

**장점:**

- 가장 공정함
- 사용자 친화적 + 악용 방지

**단점:**

- 구현 복잡도 높음

## 🏗️ 아키텍처 설계

### 방안 A: 멤버십 내부에서 혜택 사용 추적 (간단)

```
┌─────────────────────────────────────┐
│  Membership Service                 │
│                                     │
│  ┌──────────────────────────────┐  │
│  │ benefit_usage_tracking       │  │
│  │ - user_id                    │  │
│  │ - contract_id                │  │
│  │ - benefit_type               │  │
│  │ - used_at                    │  │
│  │ - reference_id (주문 ID 등)  │  │
│  └──────────────────────────────┘  │
│                                     │
│  환불 판단 시:                       │
│  SELECT COUNT(*) FROM benefit_usage │
│  WHERE contract_id = ? AND          │
│        used_at > contract.created_at│
└─────────────────────────────────────┘
```

**장점:**

- 빠른 조회
- MSA 독립성 유지
- 구현 간단

**단점:**

- 혜택 사용 이벤트를 멤버십에 기록해야 함
- 데이터 중복 (주문 서비스에도 있음)

---

### 방안 B: 메두사(주문)에서 조회 (MSA 순수성)

```
┌──────────────────┐      ┌──────────────────┐
│  Membership      │      │  Medusa (주문)   │
│                  │      │                  │
│  환불 판단 시:    │─────▶│  GET /orders     │
│  "이 사용자가     │      │  ?userId=xxx     │
│   혜택 사용했나?" │◀─────│  &createdAfter=  │
│                  │      │   {contract_date}│
└──────────────────┘      └──────────────────┘

응답:
{
  "orders": [
    {
      "id": "order_123",
      "membershipDiscountApplied": true,
      "createdAt": "2025-10-17T10:30:00Z"
    }
  ]
}
```

**장점:**

- MSA 순수성 유지
- 단일 진실 공급원 (Single Source of Truth)
- 데이터 중복 없음

**단점:**

- 네트워크 호출 필요 (지연 시간)
- 메두사 장애 시 환불 판단 불가
- 복잡도 증가

---

### 방안 C: 이벤트 기반 (비동기)

```
┌──────────────────┐      ┌──────────────────┐
│  Medusa (주문)   │      │  Membership      │
│                  │      │                  │
│  주문 생성 시:    │      │                  │
│  - 멤버십 할인    │─────▶│  이벤트 수신:    │
│    적용됨        │ Event │  BENEFIT_USED    │
│                  │      │                  │
│                  │      │  ┌─────────────┐ │
│                  │      │  │ benefit_    │ │
│                  │      │  │ usage_log   │ │
│                  │      │  └─────────────┘ │
└──────────────────┘      └──────────────────┘
```

**장점:**

- MSA 독립성 + 데이터 로컬 보유
- 빠른 조회
- 확장 가능

**단점:**

- 이벤트 인프라 필요
- 이벤트 유실 가능성 (재시도 필요)

## 💡 추천 방안

### **방안 C (이벤트 기반) + 옵션 3 (하이브리드)**

#### 이유:

1. **MSA 순수성 유지**: 각 서비스가 독립적으로 동작
2. **빠른 환불 판단**: 로컬 DB 조회로 즉시 판단
3. **공정한 정책**: 시간 + 혜택 사용 여부 모두 고려
4. **확장 가능**: 향후 다른 혜택 추가 용이

### 2. 이벤트 핸들러

```typescript
// apps/membership/src/services/benefit-usage-event-handler.service.ts

@Injectable()
export class BenefitUsageEventHandler {
  constructor(private readonly dbService: DbService) {}

  /**
   * 메두사에서 주문 생성 이벤트 수신
   */
  async handleOrderCreated(event: {
    orderId: string;
    userId: string;
    membershipDiscountApplied: boolean;
    discountAmount: number;
    createdAt: string;
  }): Promise<void> {
    if (!event.membershipDiscountApplied) {
      return; // 멤버십 혜택 미사용
    }

    // 현재 활성 계약 조회
    const contract = await this.contractReader.findActiveContract(event.userId);
    if (!contract) {
      this.logger.warn(`Active contract not found for user ${event.userId}`);
      return;
    }

    // 혜택 사용 기록
    await this.dbService.db.insert(schema.benefitUsageLog).values({
      id: generateId(),
      userId: event.userId,
      contractId: contract.id,
      benefitType: 'DISCOUNT',
      referenceId: event.orderId,
      referenceType: 'ORDER',
      amount: event.discountAmount,
      usedAt: new Date(event.createdAt),
    });

    this.logger.log(`Benefit usage logged: ${event.orderId}`);
  }
}
```

### 3. 환불 정책 업데이트

```typescript
// apps/membership/src/services/subscription/subscription-cancellation.manager.ts

/**
 * 환불 가능 여부 판단 (재구독 고려)
 */
async checkRefundEligibility(
  contract: Contract,
  plan: Plan,
): Promise<RefundEligibility> {
  // 1. 무료 체험 기간 확인 (기존 로직)
  if (this.isInTrialPeriod(contract, plan)) {
    return {
      eligible: true,
      reason: '무료 체험 기간 중 취소',
      amount: plan.price,
    };
  }

  // 2. 재구독 후 24시간 이내 + 혜택 미사용 확인 (신규)
  const isRecentResubscription = await this.isRecentResubscription(contract);
  if (isRecentResubscription) {
    const hasBenefitUsage = await this.hasBenefitUsage(contract.id);

    if (!hasBenefitUsage) {
      return {
        eligible: true,
        reason: '재구독 후 24시간 이내 + 혜택 미사용',
        amount: plan.price,
      };
    } else {
      // 혜택 사용한 경우 부분 환불 또는 환불 불가
      const usedAmount = await this.calculateUsedBenefitAmount(contract.id);
      const refundAmount = Math.max(0, plan.price - usedAmount);

      return {
        eligible: refundAmount > 0,
        reason: '재구독 후 혜택 사용 - 부분 환불',
        amount: refundAmount,
      };
    }
  }

  // 3. 환불 불가
  return {
    eligible: false,
    reason: '환불 가능 기간이 지났습니다',
    amount: 0,
  };
}

/**
 * 재구독 후 24시간 이내인지 확인
 */
private async isRecentResubscription(contract: Contract): Promise<boolean> {
  const hoursSinceCreation = differenceInHours(new Date(), contract.createdAt);

  // 24시간 이내가 아니면 false
  if (hoursSinceCreation >= 24) {
    return false;
  }

  // 이전 구독 이력이 있는지 확인 (재구독 여부)
  const allContracts = await this.contractReader.findContractsByUserId(
    contract.userId,
  );

  // 현재 계약 제외하고 이전 계약이 있으면 재구독
  return allContracts.length > 1;
}

/**
 * 혜택 사용 여부 확인
 */
private async hasBenefitUsage(contractId: string): Promise<boolean> {
  const [usage] = await this.dbService.db
    .select()
    .from(schema.benefitUsageLog)
    .where(eq(schema.benefitUsageLog.contractId, contractId))
    .limit(1);

  return !!usage;
}

/**
 * 사용한 혜택 금액 계산
 */
private async calculateUsedBenefitAmount(contractId: string): Promise<number> {
  const usages = await this.dbService.db
    .select()
    .from(schema.benefitUsageLog)
    .where(eq(schema.benefitUsageLog.contractId, contractId));

  return usages.reduce((sum, usage) => sum + (usage.amount || 0), 0);
}
```

describe('재구독 환불 정책', () => {
  it('✅ 재구독 후 10초 만에 취소 + 혜택 미사용 → 전액 환불', async () => {
    // 1. 첫 구독 → 취소
    await subscriptionService.createSubscription(userId, planId);
    await cancellationService.cancelSubscription(userId, 'TRIAL_PERIOD');

    // 2. 재구독
    await subscriptionService.createSubscription(userId, planId);

    // 3. 10초 후 취소 (혜택 미사용)
    const result = await cancellationService.cancelSubscription(
      userId,
      'CHANGED_MIND',
    );

    expect(result.refundEligible).toBe(true);
    expect(result.refundAmount).toBe(9900);
  });

  it('✅ 재구독 후 1시간 만에 취소 + 혜택 사용 → 부분 환불', async () => {
    // 1. 재구독
    await subscriptionService.createSubscription(userId, planId);

    // 2. 주문 생성 (멤버십 할인 1000원 사용)
    await benefitUsageHandler.handleOrderCreated({
      orderId: 'order_123',
      userId,
      membershipDiscountApplied: true,
      discountAmount: 1000,
      createdAt: new Date().toISOString(),
    });

    // 3. 1시간 후 취소
    const result = await cancellationService.cancelSubscription(
      userId,
      'CHANGED_MIND',
    );

    expect(result.refundEligible).toBe(true);
    expect(result.refundAmount).toBe(8900); // 9900 - 1000
  });

  it('✅ 재구독 후 25시간 후 취소 → 환불 불가', async () => {
    // 1. 재구독
    const contract = await subscriptionService.createSubscription(
      userId,
      planId,
    );

    // 2. 시간 조작 (25시간 전으로)
    await dbService.db
      .update(schema.subscriptionContracts)
      .set({ createdAt: subHours(new Date(), 25) })
      .where(eq(schema.subscriptionContracts.id, contract.contractId));

    // 3. 취소
    const result = await cancellationService.cancelSubscription(
      userId,
      'CHANGED_MIND',
    );

    expect(result.refundEligible).toBe(false);
  });
});

## 🎯 결론

### 추천 구현 순서:

1. **Phase 1**: 시간 기반 환불만 구현 (간단)
   - 재구독 후 24시간 이내 → 전액 환불
   - 혜택 사용 여부 무시

2. **Phase 2**: 이벤트 인프라 구축
   - 메두사 → 멤버십 이벤트 전송
   - `benefit_usage_log` 테이블 추가

3. **Phase 3**: 하이브리드 정책 적용
   - 시간 + 혜택 사용 여부 모두 고려
   - 부분 환불 로직 추가

### MSA 관점:

- **방안 C (이벤트 기반)** 추천
- 각 서비스가 독립적으로 동작하면서도 필요한 데이터는 이벤트로 동기화
- 메두사 장애 시에도 멤버십 서비스는 정상 동작 (로컬 데이터 기반)

이렇게 하면 사용자 친화적이면서도 악용을 방지할 수 있습니다!
