# 무료 체험 악용 방지 분석

## 📊 현재 상황

### 문제점

**무료 체험 기간 중 취소 후 재구독 시 무료 체험이 다시 적용됨**

```typescript
// subscription.creator.ts - 현재 로직
const endsAt = addDays(
  startsAt,
  plan.durationDays + (plan.trialDays || 0), // ❌ 항상 trialDays 적용
);
const billingDate = addDays(startsAt, plan.trialDays || 0);
```

### 악용 시나리오

1. 사용자가 구독 시작 (7일 무료 체험)
2. 6일차에 구독 취소 (전액 환불)
3. 다시 구독 시작 (7일 무료 체험 재적용) ← **문제!**
4. 무한 반복 가능

## ✅ 해결 방안

### 방안 1: 이벤트 소싱 기반 체험 이력 확인 (추천)

**장점:**

- 이미 구현된 이벤트 소싱 패턴 활용
- 취소된 구독도 이력으로 남음
- 정확한 체험 여부 판단 가능

**구현:**

```typescript
// subscription.creator.ts
async createNewSubscription(userId: string, plan: Plan, tier: Tier) {
  // 1. 과거 구독 이력 확인
  const hasSubscribedBefore = await this.hasUserSubscribedBefore(userId);

  // 2. 무료 체험 적용 여부 결정
  const effectiveTrialDays = hasSubscribedBefore ? 0 : (plan.trialDays || 0);

  // 3. 날짜 계산
  const endsAt = addDays(startsAt, plan.durationDays + effectiveTrialDays);
  const billingDate = addDays(startsAt, effectiveTrialDays);

  // ...
}

// 과거 구독 이력 확인
private async hasUserSubscribedBefore(userId: string): Promise<boolean> {
  const contracts = await this.contractReader.findContractsByUserId(userId);
  return contracts.length > 0;
}
```

## 🎯 권장 방안: 방안 1 (이벤트 소싱 기반)

### 이유

1. ✅ **이미 구현된 인프라 활용**
   - `subscriptionContracts` 테이블에 모든 이력 저장됨
   - `findContractsByUserId()` 메서드 이미 존재

2. ✅ **정확한 판단**
   - 취소된 구독도 이력으로 남음
   - CANCELLED, EXPIRED 상태 모두 확인 가능

3. ✅ **확장 가능**
   - 향후 "N일 후 재가입 시 체험 가능" 같은 정책 추가 용이
   - 체험 사용 횟수 제한 등 추가 정책 구현 가능

### Step 2: SubscriptionCreator 수정

```typescript
async createNewSubscription(userId: string, plan: Plan, tier: Tier) {
  return await this.dbService.db.transaction(async (tx) => {
    // 1. 첫 구독 여부 확인
    const isFirstTime = await this.contractReader.isFirstTimeSubscriber(userId);

    // 2. 무료 체험 적용 여부 결정
    const effectiveTrialDays = isFirstTime ? (plan.trialDays || 0) : 0;

    // 3. 날짜 계산 (effectiveTrialDays 사용)
    const now = new Date();
    const startsAt = now;
    const endsAt = addDays(startsAt, plan.durationDays + effectiveTrialDays);
    const billingDate = addDays(startsAt, effectiveTrialDays);
    const nextBillingDate = addDays(billingDate, plan.durationDays);

    // 4. 계약 생성 시 체험 적용 여부 기록
    const [contract] = await tx
      .insert(schema.subscriptionContracts)
      .values({
        userId,
        planId: plan.id,
        billingDate: billingDate.toISOString().split('T')[0],
        nextBillingDate: nextBillingDate.toISOString().split('T')[0],
        // 메타데이터에 체험 적용 여부 기록 (선택사항)
      })
      .returning();

    // 5. 이벤트에 체험 적용 여부 기록
    await this.contractEventManager.addEvent(
      tx,
      contract.id,
      'CREATED',
      {
        planId: plan.id,
        billingDate: billingDate.toISOString().split('T')[0],
        trialDays: plan.trialDays || 0,
        effectiveTrialDays, // ✅ 실제 적용된 체험 기간
        isFirstTimeSubscriber: isFirstTime, // ✅ 첫 구독 여부
      },
      'USER',
      userId,
      batch.id,
      userId,
    );

    // ...
  });
}
```
