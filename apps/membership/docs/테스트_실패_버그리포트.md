## ❌ 실패한 테스트 분석

### 1. 중복 취소 시도 - 에러 (Task 3)

**테스트 위치**: Line 296-303

**예상 동작**:

```typescript
await expect(
  cancellationService.cancelSubscription(testUserId, 'TRIAL_PERIOD'),
).rejects.toThrow('Contract already cancelled');
```

**실제 동작**:

```
Error: "Active subscription not found"
```

**원인 분석**:

```typescript
// subscription-cancellation.service.ts:56
const data = await this.contractReader.findContractWithPlan(userId);
if (!data) throw new Error('Active subscription not found'); // ← 여기서 에러 발생
```

**문제점**:

- `findContractWithPlan()`은 **ACTIVE 상태의 계약만** 조회
- 첫 번째 취소 후 계약 상태가 `CANCELLED`로 변경됨
- 두 번째 취소 시도 시 ACTIVE 계약을 찾지 못함
- "Contract already cancelled" 에러가 아닌 "Active subscription not found" 에러 발생

**해결 방안**:

1. **Option A**: `findContractWithPlan()`을 `findContractById()`로 변경하여 상태 무관하게 조회
2. **Option B**: 취소된 계약도 조회할 수 있는 별도 메서드 추가
3. **Option C**: 에러 메시지 검증을 "Active subscription not found"로 변경 (임시 방편)

**권장**: Option A - 계약 ID로 조회 후 상태 검증

### 2. 재구독 시 무료 체험 미적용 (Task 8)

**테스트 위치**: Line 526-569

**예상 동작**:

- 첫 구독 → 취소 → 재구독 시 무료 체험 미적용 확인

**실제 동작**:

```
Timeout: Exceeded timeout of 5000 ms for a test
```

**원인 분석**:

1. **DB 작업 지연**: 구독 생성 → 취소 → 재구독 과정이 5초 초과
2. **Cleanup 작업 지연**: 테스트 종료 시 FK 제약 조건으로 인한 cleanup 실패
3. **트랜잭션 대기**: 이전 트랜잭션이 완료되지 않아 대기

**증거**:

```typescript
// 1. 첫 구독
await subscriptionService.createSubscription(newUserId, testPlanId);

// 2. 취소
await cancellationService.cancelSubscription(
  newUserId,
  'TRIAL_PERIOD',
  '체험 후 결정',
);

// 3. 재구독 ← 여기서 타임아웃
await subscriptionService.createSubscription(newUserId, testPlanId);
```

**해결 방안**:

1. **타임아웃 증가**: `it('...', async () => {...}, 10000)` - 10초로 증가
2. **Cleanup 순서 수정**: FK 제약 조건 고려한 삭제 순서
3. **트랜잭션 격리**: 각 작업 후 명시적 대기 추가

**권장**: 타임아웃 10초로 증가 + Cleanup 순서 수정
