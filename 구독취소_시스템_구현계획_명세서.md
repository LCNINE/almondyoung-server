## 8. 비즈니스 로직 상세

### 8.1 Lazy Expiration 로직

```typescript
async checkAndUpdateSubscription(userId: string): Promise<boolean> {
  const today = new Date().toISOString().split('T')[0];

  const entitlement = await this.getActiveEntitlement(userId);
  if (!entitlement) return false;

  // 만료 체크
  if (entitlement.endsAt < today) {
    await this.expireEntitlementWithEvent(entitlement.id, userId);
    return false;
  }

  return true;
}
```

### 8.2 통합 취소 로직

```typescript
async cancelSubscription(userId: string, reasonCode: string, reasonText?: string) {
  // 1. 활성 구독 체크 (만료 시 자동 처리)
  const hasActive = await this.entitlementService.checkAndUpdateSubscription(userId);
  if (!hasActive) {
    throw new Error('No active subscription found');
  }

  // 2. 환불 가능 여부 판단
  const contract = await this.getActiveContract(userId);
  const plan = await this.getPlan(contract.planId);
  const isRefundEligible = this.checkRefundEligibility(contract, plan);

  if (isRefundEligible) {
    // 즉시 취소 + 환불
    return this.cancelImmediately(userId, reasonCode, reasonText);
  } else {
    // 정기결제 중단
    return this.cancelRecurringPayment(userId, reasonCode, reasonText);
  }
}
```

### 8.3 상태 전이도

```
ACTIVE 구독 (auto_renewal=true)
├── 환불 가능 → CANCELLED (즉시 권한 박탈 + 환불)
└── 환불 불가 → ACTIVE (auto_renewal=false, 기간 만료 시 자동 EXPIRED)

만료된 구독 (endsAt < today)
└── 자동 → isCurrent=false (Lazy Expiration)
```

## 9. 성능 최적화

### 9.1 Lazy Expiration 성능 분석

- **대상**: 1,000만 활성 구독자
- **일일 만료**: 약 27,000명
- **총 DB 부하**: 기존 대비 50% 절약
- **공평성**: 모든 사용자 만료일 정확히 적용

### 9.2 필수 인덱스

```sql
-- Lazy Expiration용
CREATE INDEX idx_entitlement_user_current_ends
ON subscription_entitlement(user_id, is_current, ends_at)
WHERE is_current = true;

-- 정기결제 중단용
CREATE INDEX idx_contracts_auto_renewal
ON subscription_contracts(user_id, auto_renewal, status)
WHERE auto_renewal = true AND status = 'ACTIVE';
```
