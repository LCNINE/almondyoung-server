# 멤버십 서비스 레이어 리팩토링 계획

## 📋 개요

현재 멤버십 서비스들이 레이어 아키텍처 규칙을 위반하고 있습니다.

- **문제**: Service가 구현 계층 역할을 하고 있음 (검증, DB 접근 등)
- **목표**: Service를 추상화 계층으로 변경 (2-3줄 오케스트레이션만)

## 🎯 리팩토링 원칙

### Service (Business Layer)

- **역할**: 2-3줄로 비즈니스 흐름만 표현
- **금지**: 검증 로직, DB 접근, 상세 구현
- **허용**: Reader/Manager 호출, 흐름 중계

### Reader (Implementation Layer)

- **역할**: 데이터 조회
- **패턴**: `{domain}-{concept}.reader.ts`
- **예시**: `entitlement.reader.ts`, `subscription-contract.reader.ts`

### Manager (Implementation Layer)

- **역할**: 검증 + 비즈니스 로직 + DB 접근
- **패턴**: `{domain}-{concept}.manager.ts`
- **예시**: `entitlement.manager.ts`, `subscription.manager.ts`

### Creator (Implementation Layer)

- **역할**: 신규 엔티티 생성
- **패턴**: `{domain}-{concept}.creator.ts`
- **예시**: `subscription.creator.ts`

## 📊 현재 서비스 분석

### ✅ 완료된 리팩토링

1. **Phase 1**: `EntitlementService` → Reader + Manager 분리 완료
2. **Phase 2**: `SubscriptionService` → Reader + Manager + Creator 분리 완료
3. **Phase 3**: `PauseService` → Reader + Manager 분리 완료
4. **Phase 4**: `PlanService` → Reader + Manager 분리 완료
5. **통합**: `CancellationReasonService` → `CancellationReasonReader`로 통합 완료

### 🔄 리팩토링 필요 서비스

1. **BenefitTrackingService**
2. **RecurringBillingService**
3. **AdminOperationsService**

#### 🔧 entitlement.service.ts (리팩토링 후)

```typescript
@Injectable()
export class EntitlementService {
  constructor(
    private readonly reader: EntitlementReader,
    private readonly manager: EntitlementManager,
  ) {}

  /**
   * ✅ 흐름만 표현: "권한 체크 → 만료 시 처리"
   */
  async checkAndUpdateSubscription(userId: string): Promise<boolean> {
    const entitlement = await this.reader.findActiveEntitlement(userId);
    if (!entitlement) return false;

    const isExpired = this.isExpired(entitlement.endsAt);
    if (isExpired) {
      await this.manager.expireEntitlement(entitlement.id, userId);
      return false;
    }

    return true;
  }

  /**
   * ✅ 흐름만 표현: "권한 조회"
   */
  async getUserEntitlement(userId: string) {
    return this.reader.getUserEntitlementDetails(userId);
  }

  /**
   * ✅ 흐름만 표현: "권한 생성"
   */
  async createEntitlement(tx, userId, tierId, startsAt, endsAt, sourceBatchId) {
    return this.manager.createEntitlement(
      tx,
      userId,
      tierId,
      startsAt,
      endsAt,
      sourceBatchId,
    );
  }

  /**
   * ✅ 흐름만 표현: "권한 조정"
   */
  async adjustEntitlement(userId, days, reason, adminId) {
    return this.manager.adjustEntitlement(userId, days, reason, adminId);
  }
}
```

```typescript
@Injectable()
export class SubscriptionService {
  constructor(
    private readonly entitlementService: EntitlementService,
    private readonly planService: PlanService,
    private readonly contractReader: SubscriptionContractReader,
    private readonly subscriptionCreator: SubscriptionCreator,
    private readonly subscriptionManager: SubscriptionManager,
  ) {}

  /**
   * ✅ 흐름만 표현: "권한 체크 → 플랜 조회 → 구독 생성"
   */
  async createSubscription(userId: string, planId: string) {
    await this.entitlementService.checkAndUpdateSubscription(userId);
    const existing = await this.entitlementService.getUserEntitlement(userId);
    if (existing) throw new ActiveSubscriptionExistsException();

    const plan = await this.planService.getPlanDetails(planId);
    if (!plan) throw new PlanNotFoundException();

    return this.subscriptionCreator.createNewSubscription(
      userId,
      plan.plan,
      plan.tier,
    );
  }

  /**
   * ✅ 흐름만 표현: "권한 체크 → 계약 조회 → 플랜 검증 → 업그레이드"
   */
  async upgradeSubscription(userId: string, newPlanId: string) {
    const current = await this.entitlementService.getUserEntitlement(userId);
    if (!current) throw new SubscriptionNotFoundException();

    const newPlan = await this.planService.getPlanDetails(newPlanId);
    if (!newPlan) throw new PlanNotFoundException();

    return this.subscriptionManager.upgradeSubscription(
      userId,
      current.contract,
      newPlan.plan,
      newPlan.tier,
    );
  }

  /**
   * ✅ 흐름만 표현: "권한 조회"
   */
  async getCurrentSubscriptionDetails(userId: string) {
    await this.entitlementService.checkAndUpdateSubscription(userId);
    return this.entitlementService.getUserEntitlement(userId);
  }
}
```

### 완료된 구조


#### 🔧 pause.service.ts (리팩토링 완료)

```typescript
@Injectable()
export class PauseService {
  constructor(
    private readonly pauseReader: PauseReader,
    private readonly pauseManager: PauseManager,
  ) {}

  /**
   * ✅ 흐름만 표현: "권한 조회 → 일시정지 실행"
   */
  async pauseSubscription(userId, startDate, endDate, reason) {
    const entitlement =
      await this.pauseReader.findActiveNonPausedEntitlement(userId);
    if (!entitlement) throw new Error('Active subscription not found');

    return this.pauseManager.startPause(
      userId,
      entitlement,
      startDate,
      endDate,
      reason,
    );
  }

  /**
   * ✅ 흐름만 표현: "권한 조회 → 일시정지 재개"
   */
  async resumeSubscription(userId) {
    const entitlement = await this.pauseReader.findPausedEntitlement(userId);
    if (!entitlement || !entitlement.pausedAt) {
      throw new Error('No paused subscription found');
    }

    return this.pauseManager.resumePause(userId, entitlement);
  }

  /**
   * ✅ 흐름만 표현: "이력 조회"
   */
  async getPauseHistory(userId) {
    return this.pauseReader.findPauseHistory(userId);
  }
}
```

#### 주요 변경사항

1. **PauseReader 생성**
   - `findPauseHistory()`: 일시정지 이력 조회
   - `findPausedEntitlement()`: 현재 일시정지된 권한 조회
   - `findActiveNonPausedEntitlement()`: 활성 상태이며 일시정지되지 않은 권한 조회

2. **PauseManager 생성**
   - `startPause()`: 일시정지 시작 (권한 연장 포함)
   - `resumePause()`: 일시정지 재개
   - 이벤트 배치 및 이력 기록 처리

3. **PauseService 간소화**
   - 모든 DB 접근 로직 제거
   - Reader/Manager 호출로 단순화
   - 2-3줄 오케스트레이션만 유지

#### 🔧 plan.service.ts (리팩토링 완료)

```typescript
@Injectable()
export class PlanService {
  constructor(
    private readonly planReader: PlanReader,
    private readonly planManager: PlanManager,
  ) {}

  /**
   * ✅ 흐름만 표현: "플랜 목록 조회"
   */
  async getAllPlans() {
    return this.planReader.findAllActivePlans();
  }

  /**
   * ✅ 흐름만 표현: "중복 검증 → 티어 생성"
   */
  async createTier(createTierInput, adminId) {
    if (await this.planReader.existsByTierCode(createTierInput.code)) {
      throw new Error(`Tier code already exists`);
    }
    if (
      await this.planReader.existsByPriorityLevel(createTierInput.priorityLevel)
    ) {
      throw new Error(`Priority level already exists`);
    }
    return this.planManager.createTier(createTierInput, adminId);
  }

  /**
   * ✅ 흐름만 표현: "티어 검증 → 플랜 생성"
   */
  async createPlan(createPlanInput, adminId) {
    const tier = await this.planReader.findTierById(createPlanInput.tierId);
    if (!tier) throw new Error('Tier not found');
    return this.planManager.createPlan(createPlanInput, adminId);
  }
}
```

#### 주요 변경사항

1. **PlanReader 생성**
   - `findAllActivePlans()`: 모든 활성 플랜 조회
   - `findPlanById()`: 플랜 상세 조회
   - `findAllTiers()`: 모든 티어 조회
   - `findPlansByTierId()`: 티어별 플랜 조회
   - `existsByTierCode()`, `existsByPriorityLevel()`: 중복 검증

2. **PlanManager 생성**
   - `createTier()`, `updateTier()`: 티어 생성/수정
   - `createPlan()`, `updatePlan()`, `deactivatePlan()`: 플랜 생성/수정/비활성화
   - 이벤트 배치 기록 처리

3. **PlanService 간소화**
   - 모든 DB 접근 로직 제거
   - 검증 로직은 Service에서 Reader 호출로 처리
   - Manager는 순수 실행만 담당
   - 2-3줄 오케스트레이션으로 단순화

```typescript
export class SubscriptionCancellationService {
  constructor(
    private readonly entitlementService: EntitlementService,
    private readonly contractReader: SubscriptionContractReader,
    private readonly cancellationManager: SubscriptionCancellationManager,
```

1. **역할 명확화**: Service가 아닌 Reader 역할
2. **도메인 응집도**: 구독 취소와 관련된 모든 기능을 한 곳에
3. **레이어 규칙 준수**: 단순 조회는 Reader로
